## 2026-03-31

### Security Audit Module — 4-phase scanner
- **secret_scanner.rs**: API keys (AWS, GitHub, Anthropic, OpenAI, Stripe, Slack, etc.), credentials, sensitive files, .env detection
- Context-aware severity: Source Code > Config > Test > Docs
- `.sparkauditignore` support with `spark audit --init` for project-aware defaults
- Google Fonts URL false positive eliminated, private key string literal detection
- **history_scanner.rs**: Git commit history scan using git2, reuses shared patterns
- **code_patterns.rs**: OWASP Top 10:2025 — SQL injection, command injection, XSS, insecure crypto, deserialization, config, path traversal
- **dep_scanner.rs**: Parses package.json/lock, requirements.txt, Cargo.toml/lock; queries OSV.dev batch API
- npm audit integration: runs automatically when package-lock.json exists
- CLI output: grouped by issue type, matched text shown per line, directory grouping for 8+ files
- `spark audit --deps`: dependency-only scan mode
- `spark audit --offline`: local-only scan (no network)
- `spark audit -o report.txt`: plain-text report output
- TUI: Audit tab with project list, severity counts, finding detail view
- 113 tests (14 secret_scanner + 10 code_patterns + 5 dep_scanner + 5 history_scanner)

### `spark certs` — Certificate scanner
- Parses .pem/.crt/.cer files with x509-parser (pure Rust)
- macOS Keychain scan via `security find-certificate`
- Home directory scan for loose key/cert/SSH files (default)
- Grouped by issuer with oldest/newest range for large groups (>5)
- Context-aware recommendations: Apple (safe to remove), Developer (renew), Self-signed (review and rotate)
- Loose file summary: counts per type + parsed cert status (expired/expiring/valid per file)
- Flags: --keychain, --expired, --summary

## 2026-03-27

### `spark doctor` — Installation health check
- Validates: binary version/path, git, repos root (exists + writable + repo count)
- Validates: config dir, config.toml, whitelist.txt
- Validates: shell integration (spark-cd in rc file, completions installed)
- Validates: platform tools (lsof, docker optional)
- Validates: status cache age (valid/expired)
- Color output with pass/fail/warn, actionable fix suggestions per failure

### CLI: `spark status` and `spark pull` commands
- `spark status [query]`: fetch + compare with remote, table with status indicators and commit age
- `spark pull <query|all>`: pull ff-only repos behind remote, exact match disambiguation
- Pull requires explicit query or "all" to avoid accidental mass operations
- Both use existing 4h status cache from TUI

### CLI: tree view and search detail
- `spark list` now shows tree grouped by host/owner/repo (was flat paths)
- `spark search` now shows status, commit age, branch, and path per result
- All CLI filters support `owner/name` format (e.g., `microsoft/skills`)

### CLI: help formatting fix
- `term_width = 80` forced on clap to prevent broken layout on wide terminals (Ghostty/Zellij)

### Performance optimizations
- **Port scanner**: batched all per-PID lookups into 2 process spawns (was N x 2)
- **Scanner container detection**: moved to post-collection using collected paths (eliminates N walkdir calls)
- **Render**: cached `group_order` in ScannerModel (eliminates per-frame O(n^2) rebuild)
- **Simplify pass**: extracted `pid_list()` helper, `rebuild_group_order` uses HashSet for O(n) dedup

### Container child actions
- Enter on a child in container detail opens `ContainerChildDetail` state
- From child detail: `c` cleans artifacts, `x` deletes repo (with confirm), `j/k` navigates siblings
- Container detail help bar updated with `[Enter] Detail`
- `CleanAllComplete` returns to child detail (not ScanResults) when cleaning from container context
